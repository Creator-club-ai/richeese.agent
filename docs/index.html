[ 01/08 ]
시장 니즈
(Market Need)
시장은 너무 작아도 안 되고,
너무 커도 안 됩니다.

너무 작으면
→ 아무리 잘해도 커지지 않고
너무 크면
→ 경쟁·변수·자본에 눌립니다.

이상적인 시장은
초기에 빠르게 장악 가능하며,
지속적으로 확장 가능한 곳입니다.
[ Case Study ]
[ 02/08 ]
작은 독점에서
시작하라
Facebook은 처음부터
"전 세계 SNS"를 노리지 않았습니다.

Harvard(작은 시장) 장악
↓
Ivy League
↓
전 세계 확장

유니콘의 시작은 '작은 독점'입니다.

[ 03/08 ]
프로덕트 훅
(Product Hook)
유니콘은 기능이 아니라
행동을 만듭니다.

질문은 이것입니다:
"설명 없이도 사용자가 반복할
단순한 행동이 있는가?"
"단순한 행동도 안 하는데 복잡한 기능은 무의미하다"
[ index® ]
[ 04/08 ]
기능이 아닌 행동
Twitter
→ 글을 쓰게 만듦
Uber
→ 버튼을 누르게 만듦
PayPal
→ 돈을 보내게 만듦

행동이 단순했기 때문에
사람들이 남았습니다.
[ index® ]
[ 05/08 ]
유통 전략
(Distribution)
"좋은 제품이면 사람들이
알아서 쓴다"는 말은
거의 항상 틀립니다.

제품만 좋다고
사용자가 저절로 찾아오지 않습니다.

유통은 마케팅이 아니라
제품 설계의 일부입니다.
[ index® ]
[ 06/08 ]
방어기제
(Moat)
소프트웨어는 쉽게 복제됩니다.
UI도, 기능도 금방 따라옵니다.

그래서 방어력이 필요합니다:
• 네트워크 효과
• 데이터
• 커뮤니티

제품은 복사할 수 있지만,
네트워크는 복사할 수 없습니다.
[ index® ]
[ 07/08 ]
실용적 조언
4가지를 모두 미리 알 필요는 없습니다.

1. 프로덕트 훅
2. 유통 전략

이 두 가지만 확실하다면
나머지(시장, 방어기제)는
성장하면서 찾을 수 있습니다.
