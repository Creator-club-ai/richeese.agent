# YouthFounderClub 콘텐츠 운영 허브

이 폴더는 디자이너 주도 제작 워크플로우를 운영하기 위한 허브입니다. AI 툴은 보조 수단으로만 사용합니다.

## 운영 원칙
- 하나의 콘텐츠는 한 명의 담당자(Responsible)와 한 명의 최종 승인자(Accountable)를 반드시 가집니다.
- 게시 전 QA 체크리스트를 통과하지 못하면 게시하지 않습니다.
- 월 1회 KPI 회고를 통해 다음 달 기획 우선순위를 조정합니다.
- 탐색형 콘텐츠는 콘텐츠 유형별 탐색 강도와 출처 신뢰도 티어를 충족해야 다음 단계로 진행합니다.
- 고난도 주제(인사이트형/논쟁형)는 `Researcher -> Planner -> Editor/Designer` 순서로 진행합니다.
- Researcher는 "추가 승인자"가 아니라 `Research Brief`를 공급하는 서비스 역할입니다.
- 슬라이드는 `6~10장`을 기본으로 하고, 정보 밀도가 높은 주제에 한해 `최대 12장`까지 확장합니다.

## 파일 안내
- `TEAM_RACI.md`: 역할/책임 매트릭스
- `CONTENT_CALENDAR_TEMPLATE.md`: 주간/월간 캘린더 템플릿
- `QA_CHECKLIST.md`: 게시 전 품질 점검표
- `KPI_RETRO_TEMPLATE.md`: 월간 성과 추적 및 회고 템플릿
- `TEAM_RACI_CURRENT_2026-03.md`: 현재 배정표(즉시 실행용)
- `CONTENT_CALENDAR_2026_W10.md`: 이번 주 캘린더(채워진 버전)
- `QA_BOARD_2026_W10.md`: 이번 주 QA 실행 보드
- `KPI_RETRO_2026-03.md`: 이번 달 KPI 기록 파일
- `SOURCE_INTAKE_PLAYBOOK.md`: 소스 제공형/탐색형 운영 규칙
- `AGENT_CALLSIGNS.md`: 에이전트 호출 이름 요약표
- `COVER_FLEX_RULES.md`: 커버 이미지 조건부/유연성 규칙
- `BRAND_GUIDE.md`: 디자이너용 브랜드/레이아웃 유연성 가이드
- `DESIGNER_HANDOFF_BRIEF.md`: 디자이너 주도 제작용 상세 핸드오프 브리프

3. "카피, 8장 슬라이드 원고 + 캡션 작성해줘"
4. "카드, 브리프 기준으로 슬라이드 디자인 완성해줘"

## 주간 운영 루틴 (권장)
1. 월요일: 캘린더 확정, 우선순위 설정
2. 화~목: 제작 및 QA
3. 금요일: 게시/성과 점검, 다음 주 리스크 정리

## 워크플로우
1. Researcher가 소스 탐색/검증 후 `Research Brief` 작성
2. Planner가 기획안 작성
3. Editor가 슬라이드/캡션 작성
4. Designer가 브리프를 바탕으로 레이아웃/비주얼을 주도적으로 재구성
5. QA 체크리스트 1차 검수
6. Designer/PD 최종 산출
7. 최종 승인자 확인 후 산출물 전달 (업로드는 오너가 수동 진행)
8. KPI 기록 및 월간 회고 반영

운영 메모:
- 디자인 시스템은 가드레일이며, 최종 시각 의사결정은 디자이너가 수행합니다.
- `slide_renderer`는 필요 시 구조 검토용으로만 사용합니다(필수 아님).

## 즉시 실행 세트 (이번 주)
1. `TEAM_RACI_CURRENT_2026-03.md` 확인 후 역할 고정
2. `CONTENT_CALENDAR_2026_W10.md` 일정 기준으로 제작 진행
3. 각 콘텐츠는 `QA_BOARD_2026_W10.md`에서 Pass 이후 오너에게 전달
4. 주간 실적은 `KPI_RETRO_2026-03.md`의 W10 행부터 누적 입력
5. 소스 입력 방식은 `SOURCE_INTAKE_PLAYBOOK.md`의 제공형/탐색형 규칙 적용

## 시작 체크리스트
- `TEAM_RACI.md`에서 실제 팀원 이름으로 소유자 지정
- `CONTENT_CALENDAR_TEMPLATE.md`를 이번 주 기준으로 채움
- 첫 게시물부터 `QA_CHECKLIST.md` 사용
- 월말에 `KPI_RETRO_TEMPLATE.md` 1회 작성
