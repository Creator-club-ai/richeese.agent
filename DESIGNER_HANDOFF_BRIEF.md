# 디자이너 핸드오프 브리프

이 문서는 고정 레이아웃 없이, 디자이너가 최종 완성도를 끌어올리기 위한 상세 브리프입니다.

## 사용 원칙
- 자동 렌더 산출물은 필요할 때만 참고 시안으로 사용합니다.
- 최종 시각 의사결정(배치, 타이포, 비주얼 강조)은 디자이너가 수행합니다.
- 팀은 레이아웃 일치가 아니라 `메시지 전달력 + 브랜드 일관성 + 시각적 완성도`로 품질을 평가합니다.

## 1) 프로젝트 정보
```markdown
- 콘텐츠 ID:
- 포맷: Instagram Carousel (1080x1350)
- 마감:
- 담당: Researcher / Planner / Editor / Designer
```

## 2) 콘텐츠 목표
```markdown
- 핵심 목표(1문장):
- 타겟 독자:
- 보고 나서 원하는 행동(CTA):
```

## 3) 핵심 메시지 우선순위
```markdown
1. 1순위 메시지:
2. 2순위 메시지:
3. 3순위 메시지:
```

## 4) 비주얼 디렉션 (디자이너 재해석용)
```markdown
- 톤: (예: editorial, modern, dynamic, warm, dark & bold 등)
- 금지: 과한 효과, 불필요한 장식, 과도한 아이콘 사용
- 컬러: BRAND_GUIDE.md의 팔레트 준수 (bg + primary text + accent)
- 강조 전략: 숫자/핵심어/행동문구에만 accent 사용
- 분위기 참조: (Unsplash 검색 키워드 또는 참고 이미지 URL)
```

## 5) 슬라이드별 의도 (역할 + 시각 패턴 조합)

기획자는 각 슬라이드의 **역할**과 **추천 시각 패턴**을 함께 전달합니다.
디자이너는 추천 패턴을 참고하되, 최종 결정은 자유롭게 합니다.

```markdown
- Slide 1:
  - 역할: Cover
  - 추천 시각 패턴: Text-first 또는 Image-first (주제 적합도 기준)
  - 필수 요소: 제목, 서브카피, 카테고리 태그
  - 기획 의도: 피드에서 무조건 시선을 멈추게 할 것
  - 이미지 키워드: (예: startup, modern office)

- Slide 2:
  - 역할: Context
  - 추천 시각 패턴: Text-first
  - 필수 요소: 공감 질문 또는 문제 제기
  - 기획 의도: "나도 그래" 라는 공감을 이끌어 낼 것

- Slide 3~N-1:
  - 역할: Body
  - 추천 시각 패턴: (Text-first / Data-List / Card-Quote 중 택)
  - 필수 요소: 메시지 1개/슬라이드
  - 기획 의도: 핵심 인사이트를 순차 전달

- Slide N-1:
  - 역할: Summary
  - 추천 시각 패턴: Card/Quote
  - 필수 요소: 핵심 요약 1~2문장
  - 기획 의도: 저장하고 싶게 만들 것

- Final:
  - 역할: CTA
  - 추천 시각 패턴: CTA-first
  - 필수 요소: 행동문구 1개
  - 기획 의도: 프로필 링크 / 저장 / 공유 유도
```

### 시각 패턴 참고표

| 패턴 | 핵심 기법 | 적합한 슬라이드 역할 |
|------|-----------|---------------------|
| 📝 Text-first | 큰 타이포, 여백, 폰트 굵기 대비, 핵심 수치 앵커 | Body, Context, Summary |
| 📸 Image-first | Unsplash 배경 + **솔리드 오버레이** (`rgba(0,0,0,0.6)`) | Cover |
| 💬 Card/Quote | 배경색 카드, 인용구 박스, 왼쪽 accent 보더 | Summary, Body |
| 📊 Data/List | Grid 레이아웃, SVG 아이콘, 번호 인덱스, 큰 수치 | Body |
| 🎨 CTA-first | 다크 배경, SVG 도트 패턴, accent 버튼 | CTA |

## 6) 데이터/근거 표현 가이드
```markdown
- 수치가 있는 슬라이드: metric/progress/timeline 형태 중 선택
- 출처 민감 정보: 본문 가독성 우선, 과도한 인용문구 금지
- 긴 문장: 2~3줄 내 재구성, 고아 단어(orphan) 방지
```

## 7) 에셋 패키지
```markdown
- 원고(JSON/문서):
- 참고 링크/이미지:
- Unsplash 이미지 키워드: (예: "startup team", "modern workspace", "technology")
- 금지 이미지/표현:
- 브랜드 참고 레퍼런스:
```

## 8) 최종 QA 기준
- [ ] 슬라이드별 메시지가 3초 내 이해되는가
- [ ] 제목/본문 대비가 과도하지 않은가
- [ ] 모바일에서 본문 가독성이 충분한가
- [ ] 브랜드 컬러 규칙을 지켰는가
- [ ] 장식보다 정보 전달이 우선인가
- [ ] **캐러셀 전체에서 최소 2~3개 시각 패턴이 혼합되었는가**
- [ ] **SVG/이미지/카드/인덱스 박스 등 구조 요소가 적절히 활용되었는가**
- [ ] **그라데이션, 글래스모피즘, 블롭 장식이 사용되지 않았는가**

## 9) 디자이너 코멘트(필수)
```markdown
- 사용한 시각 패턴 조합:
- 어떤 레이아웃 결정을 했는지:
- 초안 대비 무엇을 바꿨는지:
- 가독성/전달력 개선 포인트:
- 사용한 CSS 기법 (grid, card, SVG, pattern 등):
```

## 빠른 운영 팁
- 고정 틀을 먼저 맞추지 말고, 슬라이드별 목적을 먼저 맞춥니다.
- Cover는 정책/리스트형이면 Text-first, 인물/브랜드형이면 Image-first를 기본값으로 둡니다.
- 연속된 슬라이드가 같은 패턴이면 지루해지므로 반드시 패턴을 번갈아 사용합니다.
- 디자이너가 1차 재구성한 뒤 QA 체크리스트로 승인합니다.
