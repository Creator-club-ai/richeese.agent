# richesse.club Brand Guide

이 문서는 richesse.club 게시물 디자인의 현재 기준입니다.
핵심 방향은 `Swiss design`, `절제된 에디토리얼 구성`, `고정된 body 템플릿 1종 반복`, `텍스트 중심`, `Pretendard 고정`, `모노크롬`입니다.

## 1) 브랜드 핵심
- richesse.club 은 감각적인 장식보다 구조와 밀도로 설득하는 브랜드다.
- 게시물은 한눈에 정리된 편집물처럼 보여야 하며, 과한 브랜드 퍼포먼스보다 조용한 완성도를 우선한다.
- 피해야 할 인상은 `화려함`, `장식성`, `감성 과잉`, `트렌디함만 앞선 그래픽`이다.

## 2) 톤 가이드
- 톤 키워드: `Swiss`, `refined`, `restrained`, `editorial`
- 문장은 짧고 정제되어야 한다.
- 마지막 슬라이드도 억지 행동 유도보다 본문 흐름 안에서 마무리되는 인사이트 쪽이 더 맞다.
- 강한 주장보다 정확한 구조와 간결한 카피가 우선이다.
- source에서 잡아온 키워드는 최종 문장에 그대로 남기지 말고, 독자가 바로 이해할 수 있는 자연스러운 한국어 문장으로 다시 써야 한다.
- richesse.club 문장에서는 원문 키워드를 라벨처럼 꽂아 넣지 말고, 그 뜻과 맥락을 풀어쓴 완성된 한국어 카피로 정리한다.
- body 슬라이드끼리는 같은 말을 반복하지 않는다. 비슷한 문장이 연속으로 나오면 하나는 지우거나, 역할을 `주장 / 설명 / 근거 / 시사점`으로 명확히 갈라야 한다.

## 3) 타이포 가이드
- 기본 폰트는 반드시 `Pretendard` 를 사용한다.
- 현재 운영 기본 body 템플릿은 `brands/richesse-club/body.template.json` 을 기준으로 한다.
- 현재 고정 body 템플릿 기준 weight는 `index/body = 500`, `headline = 800` 이다.
- 현재 고정 body 템플릿 기준 body 카피 크기는 `29px` 이다.
- body 슬라이드 헤드라인은 자간을 약간 조여 `-0.04em ~ -0.06em` 범위에서 맞춘다.
- body 카피는 행간과 자간을 함께 본다. 현재 운영 기본값은 `line-height 1.714`, `letter-spacing -0.02em` 이다.
- 숫자/강조 처리도 폰트 무게, 크기, 정렬 차이로 해결한다.
- 다른 서체 혼용은 기본적으로 금지한다.
- 예외는 `cover tag` 와 `body 하단 source tag` 뿐이며, 둘 다 `Neue Haas Grotesk Display Pro Light` 를 사용한다.
- Cover tag 기본값은 `35px`, `Extra Light` 이다.
- body 하단 source tag 기본값은 최종 렌더 기준 약 `33.333px`, `Light` 이다.

## 4) 컬러 가이드
- 기본 방향은 흑백과 회색의 절제된 팔레트다.
- 슬라이드 한 장에 컬러 역할은 최대 2개를 넘기지 않는다.
- 과한 그라데이션, 화려한 색 전환, 감정적인 색 사용을 피한다.

## 5) 비주얼 원칙
- 기본 구성은 `텍스트 중심의 고정 body 템플릿 1종`이다.
- 아이콘, 일러스트, 장식 패턴, 카드형 정보 박스, 데이터 테이블을 richesse 기본 언어로 사용하지 않는다.
- 선, 정렬, 여백 정도의 최소 구조만 허용한다.
- 사진이나 영상 스틸은 현재 richesse 운영 기본값이 아니다. 별도 요청이 있을 때만 override 로 사용한다.
- Cover를 제외한 `2~N` 슬라이드는 웬만하면 같은 구조를 반복한다. richesse.club 의 본문 디자인은 변주보다 고정된 리듬이 우선이다.
- 커버에는 번호를 굳이 넣지 않는다. 번호가 필요하면 본문 시작 슬라이드부터 붙인다.

## 6) 레이아웃 방향
- Swiss design 기준으로 정렬, 간격, 비례를 엄격하게 본다.
- 모든 요소는 그리드 안에서 정리되어야 한다.
- 화면에 들어가는 정보량은 적어 보이더라도, 읽었을 때 밀도가 있어야 한다.
- 장식 박스를 많이 쓰는 대신, 타이포와 정렬로 긴장감을 만든다.
- Cover는 피드에서 시선을 멈추게 하는 역할이므로 구성에 약간의 자유를 둘 수 있다. 번호 마커는 여기서 생략해도 된다.
- Cover의 텍스트 안전 영역은 항상 `1080x1080` 기준 안에서 해결한다. 즉 1080x1440 캔버스여도, 태그와 제목은 상단 정사각형 safe area 안에 모두 들어와야 한다.
- Cover의 제목은 항상 `2줄`로 고정한다. 한 줄 처리나 세 줄 이상 분해는 richesse.club 기본값으로 허용하지 않는다.
- Cover tag는 좌상단 `[richesse.club]`, 우상단 `[인물명/주체명]` 구조를 기본값으로 두고, 둘 다 같은 baseline 안에서 정렬한다.
- Cover tag와 제목은 모두 `1080x1080` safe area 밖으로 밀지 않는다. 하단 여분 공간은 배경 이미지와 여백으로 남겨도 된다.
- 반대로 `2~N` 슬라이드는 아래 순서를 사실상 고정 레이아웃으로 사용한다: `번호 마커 + 가는 구분선 -> 헤드라인 -> lead 본문 -> secondary 본문/quote -> bullet -> 하단 좌우 source tag`.
- 현재 운영 기준 캔버스는 `1080x1440` 이며, body 템플릿은 `1080x1350` 기준 시안을 세로로 늘리지 않고 하단 여백을 확장하는 방식으로 맞춘다.
- 번호 마커는 `[01]`처럼 구조를 잡는 기준점이 되도록 충분한 존재감으로 두되, 장식처럼 튀게 만들지는 않는다.
- 현재 운영 기본 body 템플릿에서 번호는 `Pretendard Medium 37.5px` 을 기준으로 한다.
- 헤드라인은 두 줄 안에서 강하게 읽혀야 하며, body 카피보다 먼저 시선을 잡아야 한다. 현재 운영 기본값은 `Pretendard ExtraBold 66.667px` 이다.
- 본문 카피는 richesse.club 의 핵심 정보층이다. body 슬라이드에서는 설명이 충분히 살아 있어야 하며, 지나치게 짧은 카피로 비워 두지 않는다.
- 본문 카피는 `의미 있는 문단 + quote block + bullet` 구조로 간다. 핵심 주장, 맥락, 해석이 같이 살아 있어야 한다.
- 서로 인접한 body 슬라이드가 같은 결론을 반복하면 안 된다. 각 슬라이드는 이전 슬라이드와 다른 정보 기능을 가져야 한다.
- `2~N` 슬라이드에서는 번호 위치, 구분선 길이, 헤드라인 시작선, 본문 시작선, 하단 source tag 축이 거의 같은 기준을 유지해야 한다.
- 텍스트가 약한 슬라이드를 억지 사진이나 장식으로 메우지 않는다. richness는 텍스트 밀도와 정렬에서 나온다.

## 7) 사진 사용 규칙
- 현재 richesse 운영 기본 body 템플릿은 `text-only` 이다.
- 사진은 body 기본값이 아니라 예외 override 로만 사용한다.
- 사진 override 가 있더라도 텍스트보다 먼저 튀면 안 된다.

## 8) 금지 요소
- Pretendard 외 서체 남용
- 아이콘 세트 남용
- 일러스트 중심 구성
- 주황색 또는 강한 accent 컬러 중심 구성
- 과한 그라데이션
- 글로우, 블러, 장식용 그림자
- 의미 없는 패턴 과다 사용
- 감성적인 슬로건 스타일 카피 남발

## 9) 디자이너 규칙
- 한 슬라이드에는 핵심 메시지 1개만 남긴다.
- 1080x1440 규격과 모바일 가독성을 우선한다.
- 미감은 `더하기`보다 `빼기`로 만든다.
- richness는 장식이 아니라 정렬과 긴장감에서 나온다.
- Cover는 별도 예외 규칙을 따른다: `제목 2줄`, `1080x1080 safe area`, `좌우 tag`, `tag 폰트 = Neue Haas Grotesk Display Pro 35 Extra Light`.
- 커버 이후 슬라이드는 매번 새 레이아웃을 만들기보다, `brands/richesse-club/body.template.json` 과 같은 본문 템플릿을 정교하게 반복하는 편이 richesse.club 에 더 맞다.
- 강조는 컬러나 효과가 아니라 `번호-제목-본문-source tag`의 계층과 간격 차이로 만든다.
- 본문 템플릿을 반복하더라도 번호 크기, 헤드라인 리듬, 본문 행간, source tag 높이는 매 슬라이드 점검해서 완성도를 맞춘다.
- 본문 분량이 부족하면 디자인이 아니라 원고를 보강해야 한다.
