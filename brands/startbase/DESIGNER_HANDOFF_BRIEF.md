# startbase Designer Handoff Brief (Working Draft)

이 문서는 `startbase` 카드뉴스를 디자이너가 재구성할 때 참고하는 working draft 브리프입니다.

## 사용 원칙
- startbase 는 정보 플랫폼이므로, 첫 판단 기준은 분위기보다 정보 전달 속도다.
- 자동 산출물은 시작점일 뿐이며, 최종 레이아웃 판단은 디자이너가 한다.
- 다만 디자이너 판단도 `정보 구조화`, `신뢰감`, `실무 가치` 안에서 움직여야 한다.

## 1) 프로젝트 정보
```markdown
- 콘텐츠 ID:
- 브랜드: startbase
- 포맷: Instagram Carousel (1080x1440)
- 콘텐츠 유형: news | curation | explainer | checklist | insight
- 마감:
```

## 2) 콘텐츠 목표
```markdown
- 핵심 목표(1문장):
- 타겟 독자:
- 보고 나서 원하는 행동:
- 이 게시물을 저장해야 하는 이유:
```

## 3) 메시지 우선순위
```markdown
1. 제일 먼저 보여줄 정보:
2. 두 번째로 이해시킬 정보:
3. 마지막에 남길 행동/판단:
```

## 4) 시각 방향
```markdown
- 톤 키워드: clear / reliable / useful / timely / structured
- 추천 패턴: Text-first / Data-List / Card-Quote 우선
- 금지 요소: 과한 장식, 불필요한 감성 이미지, 과장된 카피 강조
- 강조 요소: 날짜, 금액, 대상 조건, 변경 포인트, 액션 포인트
```

## 5) 슬라이드별 메모

```markdown
- Slide 1:
  - 역할: Cover
  - 기획 의도: 무슨 정보인지 즉시 이해시키기
  - 필수 요소: 주제, 시점 또는 카테고리, 핵심 변화 포인트

- Slide 2:
  - 역할: Context
  - 기획 의도: 왜 지금 알아야 하는지 설명
  - 필수 요소: 배경, 변경 이유, 독자 연관성

- Slide 3~N-1:
  - 역할: Body
  - 기획 의도: 정보 1개씩 빠르게 이해시키기
  - 필수 요소: 리스트, 조건, 수치, 사례, 체크포인트 중 해당 요소

- Final:
  - 역할: CTA
  - 기획 의도: 저장 / 공유 / 원문 확인 / 다음 액션 유도
  - 필수 요소: 행동문구 1개
```

## 6) 정보 표현 규칙
- 날짜, 금액, 대상 조건은 본문 안에 묻히지 않게 분리한다.
- 출처가 중요한 포스트는 source box 또는 짧은 출처 메모를 남긴다.
- 긴 문장은 2~3줄 이내로 압축하고, 필요한 경우 리스트로 바꾼다.
- 여러 정보를 한 슬라이드에 억지로 넣지 않고, 나눌 가치가 있으면 분할한다.

## 7) 디자이너 체크리스트
- [ ] 뉴스룸/정보보드 같은 구조감이 느껴지는가
- [ ] 핵심 정보가 첫 시선에 들어오는가
- [ ] 독자가 저장할 이유가 명확한가
- [ ] 수치/날짜/대상 조건이 잘 보이는가
- [ ] startbase 톤이 과장형이 아니라 정보형으로 유지되는가
