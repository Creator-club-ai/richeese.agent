# YFC Brand Guide (Carousel)

이 문서는 고정 레이아웃을 강제하지 않고, 디자이너가 주도적으로 완성할 수 있도록 `YFC 브랜드 가드레일`을 제공합니다.

## 1) 운영 원칙
- 초안은 출발점이며, 최종 완성은 디자이너 판단을 우선합니다.
- 브랜드 일관성(톤/타이포/컬러)은 유지하고, 레이아웃은 슬라이드별로 유연하게 바꿉니다.
- 한 슬라이드에는 핵심 메시지 1개만 남기고, 나머지는 정보 계층으로 정리합니다.

## 2) 타이포 시스템
- 기본 폰트: `Pretendard`
- 권장 weight: `500 / 700 / 800`
- 본문 최소: `36px` 이상, 권장 `44px~52px`
- 본문 줄간격: `1.45~1.6`
- 제목-본문 대비: 과도한 격차를 피하고 1.3x~1.8x 안에서 조정

## 3) 컬러 사용 규칙
- 기본 팔레트: `Background(밝은 중립) + Primary Text + Accent(주황)`
- Accent는 핵심 포인트(태그/불릿/강조문) 중심으로 제한 사용
- 슬라이드 1장당 컬러 역할은 최대 3개를 넘기지 않음
- 대비가 낮은 회색 텍스트는 모바일에서 1회 이상 실기기 확인
- 다크 모드 슬라이드에서는 배경 `#1A1A1A` + 텍스트 `#FFFFFF` / `#AAAAAA` 사용

## 4) 슬라이드 역할 분류

캐러셀의 흐름상 각 슬라이드는 아래 역할 중 하나를 수행합니다:

| 역할 | 목적 | 핵심 질문 |
|------|------|-----------|
| **Cover** | 후킹 - 피드에서 시선을 멈추게 함 | "이거 뭐지?" 를 유발하는가? |
| **Context** | 도입/공감 - 독자의 문제를 끄집어 냄 | 독자가 "맞아, 나도 그래" 라고 느끼는가? |
| **Body** | 핵심 메시지 전달 (1슬라이드=1메시지) | 3초 내에 핵심이 읽히는가? |
| **Summary** | 앞 내용 요약/인사이트 정리 | 이 슬라이드만 봐도 전체가 정리되는가? |
| **CTA** | 행동 유도 (저장/공유/프로필 링크) | 독자가 무엇을 해야 하는지 명확한가? |

## 5) 시각 표현 패턴 (Visual Pattern)

역할과 별개로, 어떤 `시각 방식`으로 표현하는지를 결정합니다. 한 캐러셀 안에서 반드시 `2~3개 이상 패턴을 혼합`하여 리듬을 만듭니다.

| 패턴 | 설명 | 추천 역할 |
|------|------|-----------|
| Text-first | 타이포그래피 중심, 여백과 폰트 크기 대비로 승부 | Body, Summary |
| Image-first | 배경 사진 전체 블리드 + 오버레이 + 텍스트 | Cover, Context |
| Card/Quote | 배경색 카드, 인용구 박스, accent 보더 | Summary, Body |
| Data/List | 리스트, 큰 수치, 타임라인 | Body |
| CTA-first | 액센트 컬러 전면 사용, 버튼 형태 강조 | CTA |

## 6) 이미지/미디어 규칙
- 정책/리스트/체크리스트형 주제의 커버는 기본적으로 `Text-first`를 우선합니다.
- 인물/브랜드/인터뷰형 주제에서만 `Image-first`를 우선 검토합니다.
- 사진은 메시지를 강화할 때만 사용합니다.
- 주제와 직접 연결되지 않으면 무리하게 사진을 넣지 않습니다.
- 이미지 사용 시 텍스트와 충돌하지 않도록 대비와 여백을 우선 확보합니다.
- 이미지 위에 텍스트를 올릴 때는 반드시 솔리드 컬러 오버레이 (`rgba(0,0,0,0.6)` 내외)를 사용합니다.
- Unsplash 무료 이미지를 적극 활용할 수 있습니다.

## 7) HTML/CSS 시각 표현 활용 가이드

디자이너는 단순 텍스트 나열이 아닌, 아래 HTML/CSS 기법을 활용하여 시각적 완성도를 높입니다.

| 기법 | 용도 |
|------|------|
| Unsplash 배경 이미지 + 솔리드 오버레이 | Cover, Image-first 패턴 |
| SVG 패턴 (도트, 그리드) | CTA 슬라이드 배경 텍스처 |
| SVG 인라인 아이콘 | 리스트 불릿, 항목 시각화, CTA |
| CSS Grid / Flexbox | 다열 레이아웃, 카드 배열 |
| 배경색 카드 | 정보 그루핑, 단계 구분 |
| 왼쪽 accent 보더 | 중요 항목 강조, 변화 포인트 |
| 번호 인덱스 박스 | 순서가 있는 항목, 단계 설명 |
| box-shadow | 카드 깊이감 |
| border-radius: 6px~16px | 카드, 태그, 버튼 |

## 8) 슬라이드별 디자인 의사결정 가이드
- 고정 틀을 맞추지 말고, 메시지 우선순위에 따라 슬라이드마다 레이아웃을 새로 결정합니다.
- 역할(Cover/Context/Body/Summary/CTA) + 시각 패턴 조합으로 결정합니다.
- 시각 요소는 정보 전달이 필요한 경우에만 사용합니다.
- 한 슬라이드에 시각적 포인트는 1~2개를 넘기지 않습니다.

## 9) 디자이너 전달 포맷
- 기획자는 슬라이드별 `목적`, `핵심 메시지`, `필수 포함 요소`, `추천 시각 패턴`을 전달합니다.
- 레이아웃/타이포/강조 방식은 디자이너가 최종 결정합니다.
- 자동 렌더 산출물은 참고 시안으로만 취급하고 그대로 확정하지 않습니다.

## 10) QA 체크리스트 (디자인)
- [ ] 제목/본문 크기 대비가 과도하지 않은가?
- [ ] 본문 텍스트가 모바일에서 쉽게 읽히는가?
- [ ] 컬러/강조가 과하지 않고 핵심에만 쓰였는가?
- [ ] 슬라이드 간 레이아웃 변화가 의도적이며 일관적인가?
- [ ] 마지막 CTA가 명확한 행동을 유도하는가?
- [ ] 컴포넌트가 정보 전달을 돕고, 장식으로만 쓰이지 않았는가?
- [ ] 캐러셀 전체에서 최소 2~3개 시각 패턴이 혼합되었는가?

## 11) 디자이너 핸드오프 규칙
- 기획자는 `초안 JSON`과 `핵심 메시지 우선순위`를 전달합니다.
- 디자이너는 이 가이드를 기준으로 레이아웃/비주얼을 최종 재구성합니다.
- 규칙보다 결과물 완성도를 우선합니다.
