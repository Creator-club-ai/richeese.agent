# Instagram Slides Generator (Editorial Curator ver.)

## 이 스킬은 언제 사용하나요?

- 사용자가 "인스타 슬라이드 만들어줘" 요청 시
- 특정 주제(Founder Mode, Minimalism 등)에 대한 **체계적인 정리와 큐레이션**이 필요할 때
- 브런치 스타일의 **논점 기반 구조화된 인사이트**를 전달하고 싶을 때

---

## 핵심 철학: "Editorial Curation"

**주제를 정리하고 전달하는 에디터/큐레이터** 역할을 수행한다.
복잡한 개념을 핵심 논점으로 나누고, 각각을 명료하게 설명한다.

---

## 전개 구조 (Structured Editorial)

**1번 슬라이드(제목)**와 **마지막 슬라이드(CTA)**는 고정.
중간의 **Body (2 ~ N-1)**는 **핵심 논점 기반 소주제**로 구조화한다.

| 슬라이드 | 역할 | 가이드라인 |
|:---:|:---:|---|
| **# 1** | **Topic Headline** | 주제를 한 문장으로. "디자인이 세상을 바꾸는 방법" |
| **# 2 ~ N-1** | **Sub-topics** | 핵심 논점별 소주제로 나누어 설명 (아래 패턴 참고) |
| **# Last** | **CTA** | "저장해두고 꺼내보세요." / @richesse.club |

### Body 구성 패턴

**패턴: 소주제 (짧게) → 본문 설명 (길고 알차게) → 예시/사례**

> ⚠️ **중요**: 소주제 제목은 간결하게, 하지만 **본문 내용은 충분히 길고 깊이 있게** 작성한다.
> 짧은 메시지만 툭툭 던지는 것이 아니라, 독자가 충분히 이해하고 공감할 수 있도록 **진심을 담아 설명**한다.

**예시:**
```
# [슬라이드 N]
1. [소주제] ← 짧고 명확하게

[Founder의 목소리로 4~6줄]
[구체적 경험과 배경]
[왜 중요한지, 무엇을 배웠는지]
[독자가 적용할 수 있는 인사이트]
```

## 톤 & 매너

### 페르소나: "Editorial Voice - Direct & Assertive" (Founder's Voice)
> 소스의 내용을 **직접적이고 단정적으로** 전달한다.
> "~는 이렇게 말했습니다" 같은 보고형은 물론, 본문에서 인물의 이름(Chesky는, Musk는)을 직접 언급하는 3인칭 서술을 **엄격히 금지**한다. 
> 대신, 내가 직접 말하는 것처럼(1인칭) 서술하거나 원칙/진리처럼 서술한다.

**톤 비교:**
- ❌ "Brian Chesky는 디자이너를 '세상을 바꾸는 사람'이라고 정의했습니다"
- ❌ "Chesky는 과감하게 뉴욕행 비행기에 올랐다." (3인칭 금지)
- ✅ "디자이너는 문제 해결사가 아니다. 세상을 바꾸는 사람이다." (직접 서술)
- ✅ "나는 그때 과감하게 뉴욕행 비행기에 올랐다." (1인칭/Founder's Voice)

**핵심:**
- **단정적**: "~이다", "~해야 한다"  
- **직접적**: 인용/보고가 아닌 직접 서술
- **교과서적**: 원칙을 가르치듯

### 작성 원칙
1.  **철저한 사실 기반 (Fact-Based)**: 모든 콘텐츠는 검증된 사실(Fact)에 기반해야 한다. 드라마틱한 연출을 위해 가상의 일화를 지어내거나 사실을 왜곡하는 것은 엄격히 금지된다.
2.  **직접 서술**: 소스의 핵심을 단정적으로 전달한다
3.  **핵심 논점 파악**: 소스에서 다루는 주요 주제/논점을 파악한다
4.  **깊이 있는 본문**: 각 슬라이드의 본문은 4~6줄 정도로 작성하여 진심과 깊이를 담는다
5.  **사례의 힘**: 추상적 개념은 반드시 구체적 사례/예시와 함께 제시한다
6.  **문장 길이 변화**: 짧은 문장과 긴 문장을 자연스럽게 섞어 리듬감을 만든다
   - 긴 문장으로 배경/맥락 설명 → 짧은 선언문(완전한 문장)으로 핵심 강조
   - 현재형, 과거형을 자연스럽게 넘나들며 리듬을 만든다
7.  **질문의 전략적 사용**: 질문은 필요할 때만 사용한다
   - 핵심 논점을 강조할 때
   - 독자의 생각을 전환시킬 때
   - 매 슬라이드마다 질문을 넣지 않는다 (과도한 질문은 피로감 유발)
8.  **자연스러운 문체**: 브런치 에세이처럼 (~이다, ~해야 한다, ~였다 등, 자연스럽게 사용)
9.  **완전한 문장 종결**: 문장은 반드시 서술어로 끝맺음한다
   - ✅ 완전한 문장 사용 ("SpaceX 초기였다.", "할 뿐이다.")

## 실행 프로세스

### 0. 리서치 모드 (토픽만 주어졌을 때)
-   **Trigger**: 사용자가 URL이나 텍스트 없이 "주제(Topic)"만 던졌을 때
-   **Action**: `search_web` 도구로 해당 주제에 대한 권위 있는 아티클, 사례를 검색한다(영미권 중심)
-   **Synthesis**: 검색된 정보 중 가장 인사이트가 있는 핵심 내용을 추려 "소스(Source)"로 정의한다

### 1. 핵심 논점 파악
-   소스 내용에서 화자/저자가 다루는 주요 주제/논점을 파악한다

### 2. 소주제별 전개
-   각 주제/논점을 하나의 소주제로 확장한다
-   소주제마다: 정의 → 설명 → 예시/사례

### 3. 슬라이드 배치
-   Slide 1: 주제 제목
-   Slide 2~N: 각 소주제별 1장
-   Last Slide: CTA

### 4. 검증 (Anti-Template Check)
-   모든 슬라이드가 똑같은 리듬인지 체크한다
-   필요시 'The Breath' (짧은 슬라이드) 추가로 리듬 조절한다

## 출력 포맷

`output/instagram/slides/[날짜]_[주제]/` 디렉토리에 저장

### 예시

완전한 예시는 **`references/example-brian-chesky.md`** 참고

**기본 구조:**
```markdown
[Topic: 주제]

# [슬라이드 1]
주제 제목

# [슬라이드 2~N]
소주제별 내용 (4~6줄)

# [슬라이드 Last]
(CTA)
@richesse.club
```
