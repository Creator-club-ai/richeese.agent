# X Thread Generator

## 이 스킬은 언제 사용하나요?

- 사용자가 "X 쓰레드 만들어줘" 요청 시
- 사용자가 "트윗 쓰레드 써줘" 요청 시
- 사용자가 "긴 트윗 시리즈로 만들어줘" 요청 시

## 참조 문서

- `.agent/config/content_analyzer.md` - 콘텐츠 분석 프레임워크
- `.agent/config/brand_guide.md` - Richesse 브랜드 가이드

---

## X 쓰레드 특성

| 항목 | 스펙 |
|------|------|
| 트윗 수 | 5~10개 권장 |
| 각 트윗 글자 수 | 280자 이내 |
| 번호 표기 | 1/n 형식 권장 |
| 마지막 트윗 | CTA + 요약 |

---

## 실행 순서

### Step 1: 콘텐츠 수집

**유튜브 영상인 경우:**
```powershell
python -m yt_dlp --no-check-certificate --skip-download --write-auto-sub --sub-lang "en,ko" --convert-subs srt -o "output/source_video.%(ext)s" "<YOUTUBE_URL>"
```

**전사 파일 / 텍스트 파일인 경우:**
```powershell
Get-Content "<FILE_PATH>" -Encoding UTF8
```

**웹 아티클인 경우:**
read_url_content 도구 사용

### Step 2: 콘텐츠 분석

`content_analyzer.md`를 참고하여 추출:

- 핵심 메시지 1개
- 주요 인사이트 5~7개 (쓰레드 본문용)
- 실행 가능한 액션 2~3개
- 기억에 남는 명언 1~2개

### Step 3: 쓰레드 구조 설계

**권장 구조 (8개 트윗):**

| 트윗 | 역할 | 예시 |
|------|------|------|
| 1 | **Hook** (시선 끌기) | "99%의 스타트업이 망하는 이유를 아는가?" |
| 2 | **맥락** (왜 중요한가) | "David Sacks가 말하는 유니콘의 조건..." |
| 3~6 | **본문** (핵심 인사이트) | 각 인사이트 하나씩 |
| 7 | **액션/적용법** | "당장 해볼 것: ..." |
| 8 | **CTA + 마무리** | "도움이 됐다면 RT, 저장해두세요!" |

### Step 4: 트윗 작성

**작성 규칙:**

1. **각 트윗 독립성**: 개별로 읽어도 이해 가능
2. **번호 표기**: `1/8`, `2/8` 형식
3. **리듬감**: 긴 트윗 → 짧은 트윗 번갈아 가며
4. **이모지**: 포인트 강조용 (과하지 않게)

### Step 5: 결과물 저장

**저장 경로:**
```
output/x/threads/[날짜]_[주제]/
├── thread.txt          ← 전체 쓰레드 (구분선으로 분리)
├── tweet_01.txt        ← 개별 트윗 파일
├── tweet_02.txt
├── ...
└── source.txt          ← 원본 출처
```

## 쓰레드 작성 가이드

### ✅ Hook (첫 트윗) 패턴

| 패턴 | 예시 |
|------|------|
| **문제 제기** | "왜 '좋은 제품'이 항상 이기지 않을까?" |
| **숫자 + 약속** | "유니콘을 만드는 4가지 조건 (쓰레드)" |
| **반전** | "투자 많이 받은 스타트업이 더 빨리 망한다" |
| **경험담** | "3번 실패 후 알게 된 창업의 진실" |

### ✅ 본문 트윗 패턴

```
[숫자]. [핵심 키워드]

[설명 2~3줄]

→ [한 줄 정리 or 예시]
```

### ✅ CTA (마지막 트윗)

```
이 쓰레드가 도움이 됐다면:

🔄 RT해서 더 많은 분들과 나눠주세요
💾 저장해두고 나중에 다시 보세요
💬 당신의 경험도 댓글로 공유해주세요

팔로우하면 매일 새로운 인사이트 📩

#richesse #기업가정신
```

### ❌ 피해야 할 것

- 모든 트윗 280자 꽉 채우기
- 번호 없이 연결하기
- 첫 트윗에서 모든 내용 요약
- 마지막 트윗에 CTA 없음
