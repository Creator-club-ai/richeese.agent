# X Post Generator

## 이 스킬은 언제 사용하나요?

- 사용자가 "트윗 만들어줘" 요청 시
- 사용자가 "X 포스트 써줘" 요청 시
- 사용자가 "트위터 글 써줘" 요청 시

## 참조 문서

- `.agent/config/content_analyzer.md` - 콘텐츠 분석 프레임워크
- `.agent/config/brand_guide.md` - Richesse 브랜드 가이드

---

## X(Twitter) 포스트 특성

| 항목 | 스펙 |
|------|------|
| 최대 글자 수 | 280자 (한글 기준) |
| 이미지 | 선택사항 (1~4장) |
| 해시태그 | 2~3개 권장 |
| 최적 게시 시간 | 평일 08:00~09:00, 12:00~13:00 |

---

## 실행 순서

### Step 1: 콘텐츠 수집

**유튜브 영상인 경우:**
```powershell
python -m yt_dlp --no-check-certificate --skip-download --write-auto-sub --sub-lang "en,ko" --convert-subs srt -o "output/source_video.%(ext)s" "<YOUTUBE_URL>"
```

**전사 파일 / 텍스트 파일인 경우:**
```powershell
Get-Content "<FILE_PATH>" -Encoding UTF8
```

**웹 아티클인 경우:**
read_url_content 도구 사용

### Step 2: 핵심 인사이트 분석

`content_analyzer.md`를 참고하여 다음을 추출:

- 핵심 메시지 1개 (한 문장)
- 가장 공유 가치가 높은 인사이트 1개
- 반응을 유발할 수 있는 요소 (숫자, 반전, 질문)

### Step 3: 트윗 작성

**작성 규칙:**

1. **280자 이내** (해시태그 포함)
2. **첫 줄이 Hook**: 스크롤을 멈추게 하는 문장
3. **가독성**: 줄바꿈으로 시각적 분리
4. **CTA**: 저장/RT/댓글 유도 (선택)

**트윗 구조:**

```
[Hook - 강렬한 첫 문장]

[본문 - 핵심 인사이트 2~3줄]

[출처/CTA]

#해시태그1 #해시태그2
```

### Step 4: 결과물 저장

**저장 경로:**
```
output/x/posts/[날짜]_[주제]/
├── tweet.txt           ← 트윗 본문
├── tweet_alt.txt       ← 대안 버전
└── source.txt          ← 원본 출처
```

## 트윗 작성 가이드

### ✅ 효과적인 Hook 패턴

| 패턴 | 예시 |
|------|------|
| **숫자** | "3번 실패한 창업가가 4번째에 달랐던 한 가지" |
| **질문** | "왜 '좋은 제품'이 항상 이기지 않을까?" |
| **반전** | "투자 많이 받은 스타트업이 더 빨리 망하는 이유" |
| **인용** | "페이스북 피터틸: '경쟁은 패자들의 게임'" |

### ✅ Richesse 해시태그

```
#richesse #기업가정신 #스타트업 #창업
```

### ❌ 피해야 할 것

- 280자 꽉 채우기 (여백 중요)
- 해시태그 남발 (3개 이하)
- 링크만 던지기 (맥락 필요)
- 과장된 표현 ("인생이 바뀝니다!")

## 예시

### 입력: David Sacks 인터뷰

### 출력:

```
좋은 제품이면 알아서 팔린다?

거의 항상 틀렸다.

유니콘을 만드는 건 4가지:
→ 작은 독점 시장
→ 단순한 반복 행동
→ 바이럴 유통
→ 방어 가능한 해자

"제품은 필요조건이지, 충분조건이 아니다"
- David Sacks

#richesse #스타트업 #기업가정신
```

**글자 수: 약 180자**
