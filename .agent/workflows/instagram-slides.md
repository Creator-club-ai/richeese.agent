# /instagram-slides 워크플로우

다양한 콘텐츠 소스(유튜브 영상, 전사 파일, 아티클)를 분석하여 인스타그램 슬라이드 콘텐츠를 생성합니다.

## 참조 문서

- `content_analyzer.md` - 다양한 입력 소스 처리 가이드
- `brand_guide.md` - Richesse 브랜드 가이드
- `design_system.md` - 타이포그래피 및 레이아웃 규격

---

## 콘텐츠 전략 (Content Strategy)

> **핵심 목표**: 바이럴이 아니라 **사고를 유도하고 저장되는 콘텐츠**

### 원본 분석 원칙
1. **핵심 논지 1개** 식별 - 독자의 기존 관점을 흔드는 내용
2. **강조 포인트 선별** - 전체 슬라이드 중 핵심만 강조 (많을수록 약해짐)

### 전개 패턴 선택

| 패턴 | 사용 조건 | 구조 |
|------|----------|------|
| **문제-해결형** | "왜 X가 안 되는가?" 있을 때 | 문제 → 잘못된 접근 → 올바른 관점 → 방법 → 예시 |
| **대비형 (A vs B)** | "X라고 생각하지만 실제로는 Y" | 일반적 믿음 → 반전 → 논리 → 새 프레임 → 증거 |
| **원칙-적용형** | 명확한 원칙/법칙이 있을 때 | 원칙 선언 → 풀이 → 예시 1 → 예시 2 → 재강조 |

### FOMO 사용 기준
- **허용**: 독자의 기존 사고가 잘못되었을 수 있음을 암시할 때, CTA 슬라이드
- **금지**: 그 외에는 차분한 선언형 소제목 유지

## 톤 & 스타일

### 주제
- 기업가정신, 스타트업, CEO 인터뷰, 비즈니스 인사이트 (Richesse.club 브랜드)

### 톤

**자연스러운 존댓말 (상황에 맞게 선택)**

| 톤 | 특징 | 예시 |
|----|------|------|
| **~습니다체** | 격식 있고 신뢰감 있는 느낌 | "창업가들은 시간을 삽니다." |
| **~해요/~죠체** | 친근하고 자연스러운 느낌 | "창업가들은 시간을 사요.", "그렇죠?" |

> **권장**: 내용에 따라 자연스러운 톤 선택. 딱딱한 느낌보다는 대화하듯이.

### 글쓰기 규칙 (필수)

1. **설명하는 느낌 ❌** - 팩트와 인사이트를 던지는 느낌으로
2. **명사로 끊기 ❌** - 완전한 문장으로 마무리
3. **리듬감** - 긴 문장 → 짧은 문장으로 완급 조절
4. **40자 제한 없음** - 자연스럽게 끊기
5. **어려운 단어 ❌** - 쉬운 한국어 사용, 필요 시 바로 뒤에 설명 추가
   - 예: "제품 훅(Hook)이란 사람들이 반복할 간단한 행동입니다"
   - 예: "PMF(제품-시장 궁합)"

### 예시

```
❌ 나쁜 예:
"오직 성장."
"이게 전부."
"부자들은 투자를 함."

✅ 좋은 예:
"그들이 보는 건 오직 성장 지표뿐이다."
"아이디어? 그건 나중에 같이 만들면 된다."
"부자들은 시간을 사는 데 돈을 쓴다."

또는 더 자연스럽게:
"그들이 보는 건 오직 성장 지표뿐이에요."
"아이디어? 나중에 같이 만들면 되죠."
```

## 슬라이드 구조

### 기본 원칙

> **장수는 고정이 아님!** 핵심 내용이 5개면 7장, 10개면 12장.

| 슬라이드 | 역할 | 내용 |
|---------|------|------|
| **1장** | 표지/훅 | 주목을 끄는 한 문장 (질문/숫자/반전) |
| **2~N장** | 본문 | 핵심 내용만 (챕터별 1장씩, 3~5줄) |
| **마지막 장** | CTA | 저장/팔로우 유도 |

### 각 슬라이드 형식

```
## [슬라이드 N: 챕터명]

**강력한 제목**

본문 내용 (3~5줄)
리듬감 있게 작성
짧은 문장으로 마무리
```

## 실행 순서

### 1. 콘텐츠 추출

**유튜브 영상인 경우:**
// turbo
```powershell
python -m yt_dlp --no-check-certificate --skip-download --write-auto-sub --sub-lang "en,ko" --convert-subs srt -o "output/instagram_source" "<YOUTUBE_URL>"
```

**전사 파일 / 아티클인 경우:**
파일을 직접 읽거나 read_url_content 도구 사용

### 2. 핵심 인사이트 추출
텍스트를 분석하여 다음을 추출:
- 핵심 인사이트 (수량 제한 없음 - 내용에 따라 유동적)
- 기억에 남는 명언/인용구
- 실용적인 액션 아이템

### 3. 슬라이드 텍스트 작성

**1장 (표지/훅):**
- 강렬한 한 문장
- 예: "부자들은 절약하지 않는다."

**본문 슬라이드 (각 3~5줄):**
```
**제목**

본문 3~5줄
리듬감 있게
짧은 문장으로 마무리
```

**마지막 장 (CTA):**
```
💾 저장해두세요

부의 인사이트
매일 업데이트

@richesse.club
```

### 4. 텍스트 파일 생성
- `[주제]_slides_formal.txt` - ~습니다체 (포멀)

### 5. 캡션 작성

```
[Hook 문장 - 강렬하게]

[본문 2~3문장]

💾 저장해두고 리마인드 하세요
---
#richesse #리쉐스 #부자마인드 #돈공부 #재테크
[추가 해시태그 5~7개]
```

### 6. 이미지 생성 (선택)
generate_image 도구로 각 슬라이드 이미지 생성 (1080x1350px)

**브랜드 가이드 참조:**
- 배경: 딥 블랙 (#0D0D0D) 또는 크림 화이트 (#FFFEF7)
- 포인트: 골드 (#FFD700)
- 폰트: Pretendard Bold (헤드라인), Regular (본문)

```
instagram_slides/

    ├── [주제]_slides_formal.txt   # ~습니다체 (포멀)

    ├── slide_01_hook.png          # (선택) 이미지
    ├── slide_02~N.png
    └── source.txt                 # 원본 텍스트
```

## 핵심 체크리스트

콘텐츠 생성 후 확인:

- [ ] 원본의 핵심 내용이 모두 포함되었는가?
- [ ] 설명하는 느낌이 아닌가?
- [ ] 명사로 끊기지 않고 완전한 문장인가?
- [ ] 리듬감이 있는가? (긴 문장 → 짧은 문장)
- [ ] 어려운 단어 없이 쉽게 읽히는가?
- [ ] 각 슬라이드가 3~5줄 분량인가?
- [ ] ~습니다체 또는 ~해요체로 자연스럽게 작성되었는가?
- [ ] 캡션에 필수 해시태그 포함되었는가?
