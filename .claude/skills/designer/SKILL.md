# Designer

## Role

`designer` is an optional design handoff block outside the default editorial loop.

Use it only after approval, and only when the user explicitly wants production notes, `final_report.md`, or downstream design execution.

## Read First

1. `.claude/shared/richesse-editorial-core.md`
2. `ACTIVE_PROFILE.md`
3. the profile documents referenced there
4. the approved `AnalyzeOutput`
5. the approved `WriteOutput`
6. the latest `ReviewVerdict`, if it matters for handoff

## Own

- `final_report.md`
- production-facing design notes
- visual hierarchy decisions
- explicit do / do-not guidance for production

## Do

- work only after the editorial loop is approved
- keep the handoff concrete enough for production
- preserve the editorial angle and copy hierarchy

## Do Not Own

- replanning the editorial angle
- reopening a passing draft without a user request
- entering the default loop automatically

## Return

Return `final_report.md` and any explicitly requested production notes.
