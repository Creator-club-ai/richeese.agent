# 현재 운영 배정표 (2026-03)

이 문서는 템플릿(`TEAM_RACI.md`)을 기반으로 즉시 운영 가능한 배정으로 채운 버전입니다.

## 현재 운영 가정
- 팀 구조: `1인 오너 + 전문 리서처 + AI 제작 라인`
- 오너(최종 책임): `dasarom4`
- 핵심 역할:
  - 리서치: `Researcher` (전문 리서처)
  - 기획: `content_planner`
  - 원고: `content_editor`
  - 카드뉴스 디자인: `slide_designer`
  - 릴스 제작: `remotion_pd`

## 호출 이름 (콜사인)
- `리드` -> `dasarom4` (최종 우선순위/승인)
- `스카웃` -> `Researcher` / `content_researcher` (소스 탐색, Research Brief)
- `플랜` -> `content_planner` (기획/구조화)
- `카피` -> `content_editor` (슬라이드 문안/캡션)
- `카드` -> `slide_designer` (캐러셀 디자인/최종 산출)
- `모션` -> `remotion_pd` (릴스 렌더링)

## 제작 트랙 분리 (필수)
- Track A: `소스 제공형` (오너가 기본 소스를 전달)
- Track B: `소스 탐색형` (리서처가 소스를 탐색)

## 트랙별 추가 역할
- `Researcher`: 소스 탐색, 신뢰도 티어 분류, 리서치 브리프 작성
- `Fact Checker`: 핵심 주장-근거 매핑 검증 (기본: `Researcher`, 보조: `content_editor`)

## RACI 정의
- R (Responsible): 실무 수행
- A (Accountable): 최종 승인
- C (Consulted): 검토/협의
- I (Informed): 공유

## 운영 규칙 (현재)
- 모든 산출물은 `dasarom4` 최종 승인 후 전달합니다.
- QA에서 Hold가 나오면 당일 슬롯을 닫고 다음 슬롯으로 이동합니다.
- KPI 회고는 매월 마지막 영업일에 30분 진행합니다.
- 탐색형 소스는 콘텐츠 유형별 최소 출처 수/신뢰도 티어를 충족하지 않으면 다음 단계로 넘기지 않습니다.
